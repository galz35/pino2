  Referencia visual para entender el ciclo completo

DÍA 1: PREVENTA EN CAMPO
============================

  [PREVENTA sale a campo]
        │
        ▼
  ┌─ Visita Cliente ─────────────────────────────┐
  │                                               │
  │  ¿Cliente tiene mora cruzada (grupo eco)?     │
  │  ├── SÍ → ❌ BLOQUEADO. No puede pedir.      │
  │  └── NO → Continúa                           │
  │                                               │
  │  Preventa arma el pedido:                     │
  │  - Selecciona productos del catálogo bodega   │
  │  - Elige presentación (Bulto/Unidad)          │
  │  - Elige nivel de precio (1-5)                │
  │  - Define: CONTADO o CRÉDITO                  │
  │                                               │
  │  ¿Precio nivel 4 o 5?                        │
  │  ├── SÍ → Status: PENDIENTE_AUTORIZACION     │
  │  │        Admin web debe aprobar              │
  │  └── NO → Status: RECIBIDO                   │
  │                                               │
  │  ¿Es pedido para supermercado del dueño?     │
  │  ├── SÍ → tipo: ABASTECIMIENTO_INTERNO       │
  │  └── NO → tipo: VENTA_ESTANDAR               │
  │                                               │
  │  ¿Distribuidora vendió y bodega entrega?     │
  │  ├── SÍ → tipo: ENTREGA_POR_CUENTA           │
  │  │        requiere_cobro = false              │
  │  └── NO → Normal                             │
  └───────────────────────────────────────────────┘
        │
        ▼
  [Pedidos se sincronizan al servidor]
  (Si offline, se guardan en SQLite con UUID)


DÍA 1 (TARDE): AUXILIAR EN OFICINA (WEB)
==========================================

  [Auxiliar ve pedidos RECIBIDOS en React]
        │
        ├── Pedidos PENDIENTE_AUTORIZACION:
        │   Admin aprueba → Pasan a RECIBIDO
        │   Admin rechaza → Pasan a CANCELADO
        │
        ▼
  ┌─ Asignación Masiva ──────────────────────────┐
  │  1. Selecciona grupo de pedidos              │
  │  2. Asigna: Rutero + Placa de Camión         │
  │  3. Define fecha de entrega                  │
  │  4. Crea registro en cargas_camion           │
  │  5. Notifica al bodeguero y al rutero        │
  └──────────────────────────────────────────────┘


DÍA 2: BODEGUERO EN ALMACÉN (FLUTTER)
========================================

  [Bodeguero abre app, ve pedidos asignados]
        │
        ▼
  ┌─ Picking (Alistamiento) ─────────────────────┐
  │  Para cada pedido:                           │
  │  1. Abre checklist de items                  │
  │  2. Recorrido por pasillos:                  │
  │     □ Coca Cola 600ml → 2 bultos + 6 unid   │
  │     □ Aceite 1L → 3 bultos                  │
  │     □ Galleta María → 15 unidades            │
  │  3. Marca cada item como ✅ alistado         │
  │  4. Si falta stock: registra envío parcial   │
  │  5. Confirma → Status: ALISTADO              │
  └──────────────────────────────────────────────┘
        │
        ▼
  ┌─ Carga al Camión ───────────────────────────┐
  │  1. Abre vista consolidada del camión        │
  │  2. Ve total de bultos + unidades sueltas    │
  │  3. Carga físicamente al vehículo            │
  │  4. Confirma carga completa                  │
  │  5. → Status: CARGADO_CAMION                 │
  │  6. → Inventario bodega SE RESTA             │
  │  7. → Inventario rutero SE INCREMENTA        │
  └──────────────────────────────────────────────┘


DÍA 3: RUTERO EN RUTA (FLUTTER)
==================================

  [Rutero abre app, ve sus entregas del día]
        │
        ▼
  ┌─ Entrega ────────────────────────────────────┐
  │  Para cada pedido en la ruta:                │
  │                                               │
  │  CASO A: VENTA_ESTANDAR + CONTADO            │
  │  → Entrega items al cliente                  │
  │  → Cobra efectivo                            │
  │  → Registra pago                             │
  │  → Status: ENTREGADO                         │
  │                                               │
  │  CASO B: VENTA_ESTANDAR + CRÉDITO            │
  │  → Entrega items al cliente                  │
  │  → NO cobra ahora                            │
  │  → Sistema genera cuenta por cobrar (8 días) │
  │  → Status: ENTREGADO                         │
  │                                               │
  │  CASO C: ABASTECIMIENTO_INTERNO              │
  │  → Entrega a supermercado del dueño          │
  │  → NO cobra (precio de costo interno)        │
  │  → Status: ENTREGADO                         │
  │                                               │
  │  CASO D: ENTREGA_POR_CUENTA                  │
  │  → Solo entrega (distribuidora ya cobró)     │
  │  → Ticket de remisión C$ 0.00                │
  │  → Status: ENTREGADO                         │
  │                                               │
  │  CASO E: RECHAZO                             │
  │  → Cliente no acepta el pedido               │
  │  → Registra motivo                           │
  │  → Items quedan en inventario del rutero     │
  │  → Status: RECHAZO_TOTAL                     │
  └──────────────────────────────────────────────┘
        │
        ▼
  ┌─ Cobro de Créditos Vencidos ─────────────────┐
  │  Facturas con 8+ días → Rutero cobra         │
  │  - Pago total o parcial                      │
  │  - Registra en collections                   │
  │  - Actualiza saldo del cliente               │
  └──────────────────────────────────────────────┘
        │
        ▼
  ┌─ Cierre de Ruta ─────────────────────────────┐
  │  Al final del día:                           │
  │  1. Resumen automático:                      │
  │     - Entregados: 10/12                      │
  │     - Rechazados: 2/12                       │
  │     - Cobros contado: C$ 4,500               │
  │     - Cobros crédito: C$ 1,800               │
  │     - Efectivo total: C$ 6,300               │
  │     - Devoluciones: C$ 450                   │
  │  2. Envía cierre al servidor                 │
  │  3. Regresa a bodega con:                    │
  │     - Efectivo recaudado                     │
  │     - Items rechazados/devueltos             │
  └──────────────────────────────────────────────┘


DÍA 3 (TARDE): AUXILIAR LIQUIDA (WEB)
========================================

  [Auxiliar recibe cierre del rutero]
        │
        ▼
  ┌─ Arqueo de Dinero ───────────────────────────┐
  │  1. Cuenta el efectivo que trae el rutero    │
  │  2. Compara vs lo que dice el sistema        │
  │  3. Registra diferencia (si hay)             │
  │  4. Status: CUADRADO o CON_DIFERENCIA        │
  └──────────────────────────────────────────────┘
        │
        ▼
  ┌─ Liquidación Final ──────────────────────────┐
  │  1. Confirma totales del día                 │
  │  2. Vincula con el arqueo                    │
  │  3. Marca todos los pedidos como LIQUIDADO   │
  │  4. Cierra la ruta oficialmente              │
  └──────────────────────────────────────────────┘
        │
        ▼
  ┌─ Devoluciones ───────────────────────────────┐
  │  Bodeguero recibe items rechazados           │
  │  1. Verifica cada item devuelto              │
  │  2. Reintegra al inventario de bodega        │
  │  3. Descuenta del inventario del rutero      │
  └──────────────────────────────────────────────┘

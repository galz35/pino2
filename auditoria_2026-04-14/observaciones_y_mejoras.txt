1. PROBLEMAS ENCONTRADOS Y RESUELTOS
================================================================================

1.1 Dashboard Master-Admin mostraba "Tiendas Activas: 0"
    Archivo: web/src/pages/master-admin/master-dashboard-page.tsx
    Causa:   La lógica saltaba (return) todas las tiendas que no tenían
             license.expiryDate configurado, contando 0 de 3 tiendas.
    Solución: Se modificó para contar tiendas sin licencia como 
              "Sin Licencia / Vencidas". Se agregó nueva tarjeta "Total Tiendas"
              que muestra el recuento real (3). Ahora hay 5 tarjetas en vez de 4.
    Estado: ✅ RESUELTO - Verificado en navegador

1.2 Bodeguero aterrizaba en /products en vez de /warehouse (Bodega)
    Archivo: web/src/lib/redirect-logic.ts (línea 40)
    Causa:   El redirect del rol 'inventory' apuntaba a /store/:id/products
             cuando el flujo principal del Bodeguero es la Bodega Kanban.
    Solución: Cambiado redirect a /store/${storeId}/warehouse para que el
              Bodeguero entre directamente a su tablero de pedidos.
    Estado: ✅ RESUELTO - Verificado en navegador

1.3 Backend no conectaba por timeout (sesión anterior)
    Archivo: backend/.env
    Causa:   La base de datos remota (190.56.16.85) tardaba más de 2s.
    Solución: Se aumentó DATABASE_CONNECTION_TIMEOUT_MS=10000
    Estado: ✅ RESUELTO

1.4 Falla de SSL con PostgreSQL remoto (sesión anterior)
    Archivo: backend/src/database/database.module.ts
    Causa:   Se intentó forzar ssl:{rejectUnauthorized:false} y el host
             remoto no soporta SSL.
    Solución: Se eliminó la configuración SSL.
    Estado: ✅ RESUELTO

5. PENDIENTES Y MEJORAS FUTURAS
================================================================================

5.1 Licencias de tiendas
    Las 3 tiendas NO tienen licencia configurada (todas aparecen como
    "Sin Licencia / Vencidas"). Se necesita crear registros de licencia
    en la tabla correspondiente para que el dashboard refleje estado real.

5.2 Productos vacíos en root
    GET /api/products retorna array vacío [] — los productos existen
    pero requieren storeId como filtro. Verificar que cada página
    que consume productos pase el storeId correctamente.

5.3 Sincronización offline del Rutero
    Si el Rutero realiza ventas sin conexión a alta velocidad,
    al reconectarse podría haber colisiones de stock. Se recomienda
    implementar un sistema de cola de eventos (ledger) para validar
    transacciones secuencialmente.

5.4 Cierre de Caja simplificado
    El cierre de caja del Rutero actualmente es un placeholder.
    Para producción se necesita: Z parcial, Z diaria, diferencias
    financieras y envío ciego a bodega.

5.5 Confusion Dispatcher vs Bodega
    "Dispatcher" en el UI es terminal de despacho local, NO es Bodega.
    La Bodega (tablero Kanban) es el módulo correcto para preparar
    camiones. Capacitar a los usuarios sobre esta diferencia.
